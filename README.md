## 🔧Control de Versiones

Este proyecto utiliza Git con una convención estructurada para los commits


```bash
git add .

git commit -m "[tipo](alcance): asunto descripcion"

git push
```
#### Tipos de commit


| Tipo             | Descripción                              |
| ----------------- | ------------------------------------------------------------------ |
| fix	 | Corrección de errores |
|test | Pruebas sin modificar lógica |
|refactor | Mejora interna del código |
|revert |  Reversión de cambios|
|build | Cambios en configuración o estructura |
|feat | Nuevas funcionalidades |
|docs | Documentación |
|style | 	Formato del código |
	
	

#### Alcance

Indica el archivo o módulo afectado.




#### Asunto

-En presente Máximo 

-50 caracteres 

-Sin punto final



#### Descripción

Explica: Por qué se hizo el cambio, cómo se implementó
## Ejemplo:
```bash
git add .
git commit -m "refactor(CuentaFragment.java): mejora formato

Se reorganiza el código para mejorar la legibilidad
y facilitar el mantenimiento"
git push
```
